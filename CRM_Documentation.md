# CRM - Technical Documentation

## 1. Project Overview
**CRM** is a premium, real-time Project Management and Payroll platform designed for internal teams to track productivity, manage project lifecycles, and automate complex financial calculations. The system focuses on high-fidelity data visualization, instant responsiveness, and administrative transparency.

---

## 2. Technology Stack
- **Frontend**: React.js (v18+) with Vite
- **Styling**: Tailwind CSS for high-performance, utility-first UI
- **Icons**: Lucide React
- **State Management**: React Context API & Custom Reactive Hooks
- **Backend/Database**: Google Firebase (Firestore, Authentication)
- **Real-time Engine**: Firestore `onSnapshot` for 0-latency UI updates
- **Deployment**: Production-ready `.htaccess` configuration for SPAs

## 3. Core Features

### 3.1 Project Management
- **Live Pipeline**: Dynamic kanban-style management of project statuses (Ongoing, Completed, Cancelled).
- **Stage Tracking**: Integrated `StageSelector` allowing admins to move projects through custom lifecycles with automated logging.
- **Assignment System**: Real-time assignment/unassignment of Workers and Admins to specific project scopes.
- **Activity Logs**: Global and project-specific audit trails for all creation, deletion, and status changes.

### 3.2 Real-time Financial Engine
- **Live Budget Preview**: Real-time cost estimation within project forms that updates instantly as workers are toggled.
- **P&L Tracking**: Automatic calculation of **Overhead Expenses**, **Labor Costs**, and **Realized Profit** based on validated work sessions.
- **Expense Management**: Categorized project-specific expense tracking with audit logs for budget changes.
- **Multi-Currency Support**: Native support for Indian Rupee (₹) formatting.

### 3.3 Payroll & Work Tracking
- **Floating Shift Timer**: A 60fps, high-performance timer for workers to log shifts against specific projects. Optimized with `requestAnimationFrame` for buttery-smooth movement.
- **Work Hour Dashboard**: Visual calendar (Responsive) for workers to track validated, pending, and active shifts.
- **Validation Workflow**: Admin-controlled "Shift Validation" queue to verify work quality before it impacts payroll.
- **Salary Engines**:
  - **Hourly**: Total Hours × Hourly Rate.
  - **Monthly**: Pro-rated based on a 20-day "Qualified Day" cycle (6+ hours/day).
  - **Project-Based**: Standard hour thresholds (e.g., 160h) with specific Overtime Rates.

### 3.4 Team Directory (People Section)
- **Worker Cards**: Compact, premium UI cards showing real-time monthly compensation and payment basis.
- **Profile Management**: Role-based access control (Super Admin, Admin, Worker).
- **Financial Settings**: Granular control over individual salary types, rates, and overtime thresholds.

## 4. Architecture & Security

### 4.1 Data Model (Firestore)
- **`users`**: Central profiles with role, salary settings, and auth metadata.
- **`projects`**: Project details, assignments, and billing information.
- **`shifts`** (Collection Group): Granular work logs with durations and validation status.
- **`stageChangeLogs`**: History of project status transitions.
- **`budgetLogs`**: History of billing and contract value modifications.

### 4.2 Security Rules
The system implements strict **Firestore Security Rules (v2)**:
- **Authenticated Access**: Only registered users can read project logs and worker summaries.
- **Admin Guard**: Destructive actions (deleting projects, updating roles, validating payroll) are strictly locked to users with `admin` or `super_admin` roles.
- **Worker Isolation**: Workers can only modify their own active shift heartbeats and read data pertinent to their assigned projects.
